# Pre-ship audit — 2026-04-20

Scope: Legalize SDK monorepo at `legalize-dev/legalize-sdks`, three SDKs (Python / Node / Go) targeting `0.1.0`.

## Executive verdict

**SHIP-WITH-ONE-BLOCKER.** Two of three SDKs are ready to tag. Python has a single real contract-violation that must be fixed before `python-v0.1.0` goes out: the retry loop retries POST/PATCH on 429/5xx by default, which risks duplicate `webhooks.create()` and duplicate delivery retries. Node and Go enforce idempotency correctly and can tag immediately. All 18 endpoints are wired in the three SDKs with the correct per-language casing, the env-var contract is byte-for-byte identical, and webhook signature verification is constant-time + verify-before-parse in all three. Zero CodeQL alerts, zero Dependabot alerts, correct secrets present, branch protection active. Several docs still describe Node/Go as "reserved" or reference the old `legalize` npm name — cosmetic, fix opportunistically.

### 2. Environment-variable contract — **PASS**

Byte-for-byte identical behavior across `python/src/legalize/_client.py:55-81`, `node/src/env.ts:28-79`, `go/env.go:24-70`. Precedence `explicit > env > default`, empty strings treated as unset, `leg_` prefix validation before first I/O. Parity tests:

Zero-config `Legalize()` / `new Legalize()` / `legalize.New()` works in all three.

### 3. Webhook signature verification — **PASS (with one API-surface divergence)**

Cryptographic correctness is identical:

- Constant-time compare in all three: `hmac.compare_digest` (`webhooks.py:161`), `crypto.timingSafeEqual` on equal-length Buffers (`webhooks.ts:128,137`), `crypto/subtle.ConstantTimeCompare` (`webhooks.go:110`).
- Verify-before-JSON-parse in all three.
- Multi-signature `v1=...,v1=...` header supported.
- Default tolerance 300 s.

**Divergence (not a ship blocker):** `WebhookVerificationError` carries a machine-readable `.reason` in Node (`errors.ts:145-167`) and Go (`errors.go:150-162`); Python intentionally elides it (`webhooks.py:136,149,159,162,168,170`) to avoid leaking which specific check failed. Either direction is defensible; pick one and reflect in PARITY.md §7.

### 4. Error hierarchy — **PASS**

All 12 error types present in each SDK with matching status-code dispatch. Confirmed line-by-line at `_errors.py:116-176`, `errors.ts:107-143`, `errors.go:78-162`:

- 400→InvalidRequestError, 401→AuthenticationError, 403→ForbiddenError, 404→NotFoundError, 422→ValidationError, 429→RateLimitError, 503→ServiceUnavailableError, 5xx→ServerError.
- `APIError` base exposes `status_code`/`code`/`body`/`response`/`request_id` (idiomatic casing per language).
- `RateLimitError` carries `retry_after` / `retryAfter` / `RetryAfter` in all three.

### 5. Retry semantics — **FAIL (blocker for Python 0.1.0)**

Python `_retry.py:53-59` `RetryPolicy.should_retry(attempt, *, status)` takes no method parameter. `_client.py:303-334` (sync) and `:425-476` (async) call it without method filtering, and the client calls `APIError.from_response` only when `should_retry` returns False. Consequence: any POST or PATCH hitting a 5xx or 429 is retried up to `max_retries=3` times. Real-world risk:

- `client.webhooks.create(url=..., events=...)` → 500 → SDK retries → server could end up with up to 3 webhook endpoints.
- `client.webhooks.retry(endpoint_id, delivery_id)` → 429 → SDK retries → duplicate delivery retries.

Node and Go do this correctly:

- Node: `retry.ts:80-96` enforces `IDEMPOTENT_METHODS`; client passes `method: upperMethod` at `client.ts:175,204`. Covered by `tests/retry.test.ts:76,280`.
- Go: `go/client.go:322,335,357` check `idempotent` before every retry; covered by `retry_test.go:181`, `client_test.go:280`.

All three honor `Retry-After` as delta-seconds and HTTP-date, cap at `max_delay`, and apply full jitter.

**Fix shape for Python**: extend `should_retry(attempt, *, status, method="GET")` with a safe-method set `{"GET","HEAD","OPTIONS","PUT","DELETE"}` (matching Node/Go), thread `method` through `_send_with_retry` and `AsyncLegalize.request`, add `test_retry.py::test_post_not_retried_by_default` to lock in parity.
